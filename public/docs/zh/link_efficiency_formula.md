### 1. 术语解释（按界面顺序）

- **Link Type（链路类型）**：Downlink（下行链路），指卫星到地面站的传输方向（与 Uplink 上行相反）。下行通常使用更低的功率和更高的编码效率，本例即典型卫星广播场景。
- **Calculation Mode（计算模式）**：Data Rate → Symbol Rate（从有用数据速率反推符号速率）。这是卫星链路规划中最常见的模式，用于已知有用比特率（例如视频流 1000 kbps）时反算发射机需要的符号速率。
- **MODCOD（调制与编码方式）**：QPSK 3/4。  
  - QPSK（Quadrature Phase Shift Keying，四相移相键控）：4 种相位，每符号携带 **m = 2** 比特。  
  - 3/4：**内码率 r**（卷积码率），表示每 4 个编码比特中有 3 个有效信息比特。
- **Roll-off Factor（α，滚降因子）**：0.05。  
  脉冲成形滤波器（平方根升余弦滤波器 SRRC）的参数。α=0 表示理想矩形频谱（无扩展），实际必须 >0 以避免码间干扰（ISI）。DVB-S 常用 0.35、0.25、0.20、0.05（新系统）。
- **Overhead Efficiency（η_oh，开销效率）**：188/204 (DVB-S RS)。  
  DVB-S 外码采用 Reed-Solomon (204,188) 码：每 204 字节码块中，**188 字节为有效 MPEG-TS 载荷**，16 字节为校验。效率 η_oh = 188/204 = 47/51 ≈ 0.92157。
- **Input Data Rate（输入数据速率）**：1000 kbps = **有用信息比特率 R_b**（即最终用户可用的比特率，如视频流速率）。
- **输出指标**：
  - Data Rate：仍为 1000.0000 kbps（输入值，未变）。
  - Symbol Rate (SR)：符号速率，单位 ksps（kilo-symbols per second）。
  - Bandwidth：占用射频带宽（RF Bandwidth）。
  - Spectral Efficiency：频谱效率，单位 bits/s/Hz。

### 2. 公式及详细推导（全部使用 KaTeX）

#### 核心公式（DVB-S 有用比特率与符号速率关系）

\[
R_b = SR \times m \times r \times \eta_{oh}
\]

- \( R_b \)：有用信息比特率（kbps）
- \( SR \)：符号速率（ksps）
- \( m = 2 \)（QPSK）
- \( r = 3/4 \)（卷积码率）
- \( \eta_{oh} = 188/204 \)（RS 开销）

**反推符号速率**（本应用模式）：

\[
SR = \frac{R_b}{m \times r \times \eta_{oh}}
\]

**精确计算**（分数形式，避免浮点误差）：

\[
m \times r \times \eta_{oh} = 2 \times \frac{3}{4} \times \frac{188}{204} = \frac{3}{2} \times \frac{47}{51} = \frac{141}{102} = \frac{47}{34}
\]

\[
SR = \frac{1000}{\frac{47}{34}} = 1000 \times \frac{34}{47} = \frac{34000}{47} = 723 + \frac{19}{47} \approx 723.404255 \, \text{ksps}
\]

→ 界面显示 **723.4043 ksps**（四舍五入正确）。

#### 占用带宽公式（Nyquist 带宽 + 滚降）

\[
B = SR \times (1 + \alpha)
\]

代入：

\[
B = 723.404255 \times 1.05 = 723.404255 \times \frac{21}{20} \approx 759.574468 \, \text{kHz}
\]

→ 界面显示 **759.5745 kHz**（精确匹配）。

#### 频谱效率公式

\[
\eta_s = \frac{R_b}{B} = \frac{m \times r \times \eta_{oh}}{1 + \alpha}
\]

\[
\eta_s = \frac{47/34}{1.05} = \frac{47}{34} \times \frac{20}{21} \approx 1.3165266 \, \text{bits/s/Hz}
\]

→ 界面显示 **1.3165**（正确舍入）。

### 3. 原理解释（DVB-S 物理层工作原理）

DVB-S 是欧洲 ETSI 卫星数字电视标准（1997 年），采用**级联纠错 + QPSK** 方案，目的是在卫星功率受限、带宽宝贵的环境下实现可靠传输：

1. **MPEG-TS 流** → 188 字节包（有效数据）。
2. **外码（RS 码）**：(204,188) 每包加 16 字节校验 → 开销 η_oh = 188/204。目的是纠正突发错误。
3. **内码（卷积码）**：码率 1/2、2/3、3/4、5/6、7/8（本例 3/4），提供前向纠错，抗高斯噪声和雨衰。
4. **QPSK 调制**：每符号 2 比特 → 结合内码与 RS 后，**每个符号的有效信息比特数** = 2 × (3/4) × (188/204)。
5. **脉冲成形**：平方根升余弦滤波器，滚降 α，使相邻符号不重叠（无 ISI），但频谱宽度扩大到 (1+α)SR。
6. **射频传输**：实际占用带宽 B = SR × (1+α)，卫星转发器以此分配频谱资源。

**为什么必须同时考虑 RS 开销**？  
如果只用内码率而不乘 188/204，最终用户看到的“有用”速率会少约 8%，无法匹配实际视频流 1000 kbps。因此本应用正确地同时乘了两个效率。

### 1. 关键变化与术语解释（对比上一个 DVB-S 示例）

| 参数                  | 上一个例子 (DVB-S)              | 这个例子 (DVB-S2/S2X)          | 说明 |
|-----------------------|----------------------------------|----------------------------------|------|
| Overhead Efficiency   | 188/204 ≈ 0.92157 (RS码)        | 1.0 (无RS)                      | DVB-S2 取消了外码RS，使用更强的LDPC内码 |
| MODCOD                | QPSK 3/4                        | QPSK 3/4                        | 相同调制+码率，但实际编码方式不同 |
| Roll-off Factor (α)   | 0.05                            | 0.05                            | 相同（DVB-S2/S2X 支持更低α，如0.05、0.10、0.15、0.20、0.25、0.35） |
| 输入有用数据率 R_b    | 1000 kbps                       | 1000 kbps                       | 相同 |
| 输出符号速率 SR       | ≈723.4043 ksps                  | 666.6667 ksps                   | 更低 → 更省带宽 |
| 带宽 B                | ≈759.57 kHz                     | 700.0000 kHz                    | 减少约8% |
| 频谱效率 η_s          | ≈1.3165 bits/s/Hz               | 1.4286 bits/s/Hz                | 提升约8.5% |

**核心差异原因**：DVB-S2/S2X 去掉了 Reed-Solomon (204,188) 外码的8%开销，全部比特都用于纠错+信息，效率更高。

### 2. 公式及精确推导（DVB-S2/S2X 模式）

#### 核心关系（无外码开销）

\[
R_b = SR \times m \times r \times \eta_{oh}
\]

- η_oh = **1.0**（DVB-S2/S2X 使用 LDPC + BCH 内码，无需额外 RS 层）
- m = 2（QPSK）
- r = 3/4（码率，实际为 LDPC 码率）

**反推符号速率**：

\[
SR = \frac{R_b}{m \times r \times \eta_{oh}} = \frac{1000}{2 \times \frac{3}{4} \times 1.0} = \frac{1000}{1.5} = \frac{2000}{3} \approx 666.6666667 \, \text{ksps}
\]

→ 界面显示 **666.6667 ksps**（完美匹配，典型浮点表示）。

#### 占用带宽（仍使用升余弦滚降）

\[
B = SR \times (1 + \alpha) = \frac{2000}{3} \times 1.05 = \frac{2000}{3} \times \frac{21}{20} = \frac{2000 \times 21}{3 \times 20} = \frac{42000}{60} = 700 \, \text{kHz}
\]

→ **700.0000 kHz**（精确整数，计算无误差）。

#### 频谱效率

\[
\eta_s = \frac{R_b}{B} = \frac{1000}{700} = \frac{10}{7} \approx 1.42857142857 \, \text{bits/s/Hz}
\]

→ 界面显示 **1.4286**（四舍五入到4位小数，正确）。

或者用参数直接表达：

\[
\eta_s = \frac{m \times r \times \eta_{oh}}{1 + \alpha} = \frac{2 \times 0.75 \times 1.0}{1.05} = \frac{1.5}{1.05} = \frac{150}{105} = \frac{10}{7} \approx 1.42857
\]

### 3. 原理对比：为什么 DVB-S2/S2X 效率更高？

| 方面               | DVB-S (1997)                          | DVB-S2 (2005) / DVB-S2X (2014)                  | 实际影响（本例） |
|--------------------|----------------------------------------|--------------------------------------------------|------------------|
| 外码               | Reed-Solomon (204,188)                | 无（或可选 BCH 外码，但通常不启用）              | 节省 ~7.84% 开销 |
| 内码               | 卷积码 + Viterbi 译码                 | LDPC（低密度奇偶校验码）+可选 BCH                | 更接近香农限，相同码率下抗噪能力强 1–2 dB |
| 调制方式           | 仅 QPSK                               | QPSK ~ 32APSK / 64APSK / 128APSK / APSK 等      | 本例仍 QPSK，但未来可更高阶 |
| 滚降因子支持       | 0.35 / 0.25 / 0.20                    | 扩展到 0.05 / 0.10 / 0.15 等                     | 本例用 0.05，已很低 |
| 典型频谱效率提升   | 基准                                  | 约 25–35%（视 MODCOD）                           | 本例仅因去 RS 就提升 ~8.5% |

**DVB-S2/S2X 的效率公式**（简化版，无 pilot、无帧头/PL 头部损耗考虑）：

- 实际工程中还会额外扣除 1–3% 的物理层帧开销（PLHEADER、pilot symbols 等），但本应用显然简化模型为 η_oh = 1.0（常见于快速估算工具）。
- 如果是严格计算，DVB-S2 的“净”效率 ≈ 0.97–0.99，但 1.0 是业界最常用的近似，尤其在 α 很低时影响很小。

\[
R_b = 1000 \times 1.382352941176 = 1382.352941176 \, \text{kbps}

### 4. 原理补充：为什么这个模式很重要？

在实际卫星通信工程中：
- 转发器（transponder）通常按固定**带宽**或固定**符号率**分配（例如 Ku 波段常见 36 MHz、54 MHz 转发器，对应符号率 ~27–45 Msps）。
- 运营商/广播商先拿到分配的符号率（如 1000 ksps 小带宽测试载波或窄带回传），然后用这个工具反算：**我能传多少有用数据？**（1382 kbps ≈ 1.38 Mbps）。
- 再根据视频编码（如 H.264/H.265）或 IP 数据需求，判断是否够用，或需要申请更大带宽/更高 MODCOD。

**注意事项**（本应用简化模型）：
- 未扣除 DVB-S 物理层帧同步、能量分散等微小开销（通常 <0.5%）。
- 未考虑 pilots、PL-header（DVB-S 无此概念，但 DVB-S2 有）。
- 实际部署中还会加入约 3–5% 的 MPEG-TS 打包/复用开销，但本工具聚焦**物理层有用速率**。

### 4. 实际工程意义（为什么这个组合最常用）

- 卫星运营商通常按**固定符号率**或**固定带宽**出租转发器容量（例如 1 Msps 小载波用于 VSAT 回传、测试、窄带广播）。
- 给定 1000 ksps，DVB-S2/S2X 可输出 **正好 1.5 Mbps** 有用速率（1500 kbps），非常干净的数字，便于规划 IP 流量、视频流、数据广播。
- 如果用老 DVB-S 标准，相同符号率只能到 ~1.382 Mbps，损失明显。
- 现代卫星（如 Eutelsat、SES、Intelsat 的 HTS 高通量卫星）几乎全部采用 DVB-S2/S2X，正是因为频谱效率提升 20–50%（视 MODCOD）。
